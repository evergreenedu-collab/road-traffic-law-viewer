# 도로교통법 개정 연혁 추적기

## 프로젝트 목적
한국도로교통공단(KoRoad) 교수진이 도로교통법 특정 조문의 개정 이력을 쉽게 파악할 수 있도록 지원하는 도구.
- 법령정보센터에서 마우스로 연혁을 일일이 찾는 불편함 해소
- **개정이유 기반 역추적**: "의무/권장 구분은 언제부터?" 같은 질문에 답변
- **공포일 vs 시행일 vs 부칙별도시행일** 구분
- **상위법 ↔ 하위법령(시행령·시행규칙) 연쇄 개정** 자동 매핑

## 사용자
- 한국도로교통공단 교통안전교육본부 교수진
- 비개발자도 사용 가능해야 함

## 기술 스택
- Python 3.10+, requests (데이터 수집)
- HTML/CSS/JS — 단일 파일, 오프라인 실행 가능 (뷰어)
- 국가법령정보 오픈API: http://www.law.go.kr/DRF/
  - API 키: `evergreen_edu`
  - 검색: `lawSearch.do` (target=law)
  - 본문: `lawService.do` (target=law)

## API XML 주요 태그
```
<기본정보>
  <법령ID>, <공포일자>, <공포번호>, <시행일자>, <제개정구분>, <소관부처>
<조문단위>
  <조문번호>, <조문가지번호>, <조문제목>, <조문시행일자>,
  <조문제개정유형>, <조문내용>
  <항> → <항번호>, <항내용>, <항제개정유형>
    <호> → <호번호>, <호내용>
<부칙단위>
  <부칙공포일자>, <부칙공포번호>, <부칙내용>
<제개정이유> → <제개정이유내용>
<개정문> → <개정문내용>
```

## 법령 그룹
| 구분 | 법령명 |
|------|--------|
| 법률 | 도로교통법 |
| 시행령 | 도로교통법 시행령 |
| 시행규칙 | 도로교통법 시행규칙 |

## 출력 형식
1. **JSON**: 구조화된 전체 데이터 (article_XX_history.json)
2. **마크다운**: 요약 표 + 상세 서술 (Claude Project 업로드용)
3. **HTML 뷰어**: JSON 드래그&드롭 → 표 + 상세 패널

## 코딩 규칙
- 주석은 **한국어**로 작성
- 변수명, 함수명은 **영어** (snake_case)
- docstring은 한국어
- 에러 메시지는 한국어 포함 (사용자가 비개발자)
- print 출력에 이모지 사용하여 진행상황 표시

## 용어 규칙 (교육본부 기준)
- 차로변경(O) / 차선변경(X)
- 보도(O) / 인도(X)
- 경음기(O), 방향지시등(O), 안전띠(O)
- 교육본부(O)

## 파일 구조
```
law_history_project/
├── CLAUDE.md              ← 이 파일
├── README.md              ← 사용 설명서
├── requirements.txt       ← Python 의존성
├── law_history_collector.py  ← 데이터 수집 스크립트
├── law_history_viewer.html   ← 결과 뷰어 (오프라인 실행)
```

## 주의사항
- API 요청 간격 최소 0.6초 유지 (서버 부하 방지)
- 판례/법령 내용은 절대 지어내지 말 것 — API 결과만 사용
- 법령명 검색 시 정확히 매칭 (예: "도로교통법"이 "도로교통법 시행령"과 혼동되지 않도록)
- Windows 경로: 공백·괄호 포함 시 따옴표 처리
  - 바탕화면: C:\Users\user\OneDrive\바탕 화면\
